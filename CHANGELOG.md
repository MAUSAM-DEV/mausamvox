# Changelog

One dated line per completed step/session. Newest first. Each entry ends with the commit hash.

## 2026-06-30

- Add a read-aloud script to the Voice Lab recording UI — users previously had no guidance on what to say while recording a clone. New `recordingScripts.ts` holds 3 phonetically-varied passages (~30–45s each, so one pass clears the 30s minimum) + a `nextScript()` rotor; RecordStep renders the active passage above the Quick/Pro capture panel (not Upload) under the label "Read this aloud (or sing across your range):" with a "New script" shuffle button. Includes explicit singing guidance (go low/high + soft/loud, don't read flatly) since clones here are often used for singing. Display-only — no backend/training changes. (commit: 61e7ad2)
- Fix marketing copy that over-promised 6-stem separation — we ship 4-stem htdemucs (guitar+piano/6-stem deferred). Corrected three landing spots: Smart Stem Studio card desc + badge (Features.tsx) "6-stem"/"6 Stems" → "4-stem"/"4 Stems"; StemSplit Engine desc (TechSection.tsx) "Clean 6-stem separation" → "Clean 4-stem separation"; Starter pricing tier (Pricing.tsx) "6-stem split + BPM/key" → "4-stem split + BPM/key". Left the unrelated Choir "SATB · 4 Stems" copy, the already-accurate Free-tier "4-stem split", and dashboard copy untouched. (commit: 76a0164)
- Fix silently-ignored Demucs param in stem-split — the cjwbw/demucs cog's stem-selection key is `model_name`, not `model`, so our `model: 'htdemucs'` was stripped by Replicate and the cog ran its default htdemucs (4-stem) by luck. Renamed to `model_name: 'htdemucs'` to make the intent explicit/correct, and dropped the equally-ignored `mp3: true` key (output_format already defaults to mp3). Output is identical — still 4-stem htdemucs, no behavior change. (Investigation also confirmed we are NOT paying 6-stem compute: the cog's output object always has 6 fixed keys, but guitar/piano come back empty under htdemucs.) (commit: 401a5fe)
- Add per-stage cold-start vs compute timing logs to the pipeline routes (instrumentation only, no behavior change). New `logReplicateTiming` helper (`src/lib/replicate-timing.ts`) logs one `TIMING` line per finished prediction — cold-start/queue (started_at−created_at), compute (metrics.predict_time), total — wired into stem-split/Demucs, karaoke-split, and voice-convert/RVC GET success branches. gender-split (MVSEP) logs wall-clock total from the client-supplied elapsedMs (MVSEP returns no queue/compute split). Read in Vercel logs by searching `TIMING` after one real swap. (commit: 73fb83c)
- Recent Swaps now saves the FULL track (clone vocal + instrumental) instead of the vocal-only file — ResultStep uploads the built mix (presign→PUT) and persist stores it as result_path; falls back to the vocal if the mix/upload fails; previews never upload. (commit: ff33ce5)
- Fix hardcoded sidebar "My Voices" badge — VSidebar + VLSidebar now render a live voice_clones count (count:exact, head, eq user_id) instead of a static '3'. (commit: 42b21ba)
- Fix stale dashboard counts — Voice Swaps + Voice Clones counts now refetch on window focus and tab visibilitychange (were fetched once on mount); delete handler refetches authoritatively instead of optimistic local −1. (commit: 3bb8d75)
- Restore duet-split warning before auto-split — declared-duet uploads no longer silently run the 250cr gender-split; the existing amber gate renders and the user must click "Split duet · 250 cr · Premium" to proceed (Option A: handleStemDone no longer auto-calls runGenderSplit / sets genderSplitting). (commit: d273e69)
