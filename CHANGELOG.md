# Changelog

One dated line per completed step/session. Newest first. Each entry ends with the commit hash.

## 2026-07-02

- Fix the "fresh Full-song swap plays the WRONG voice (different each attempt) and is missing music" bug — root-caused to the `mvox_stem_session` localStorage cache restoring **silently on every page mount** (UploadStep auto-flips to "done", so a restored track looks like a fresh swap) combined with only the full vocal stem having a durable re-signable path. Two failure legs: (1) the lead vocal — the DEFAULT input to every single-voice swap via karaoke-split — carried no `vocalsPath`, so voice-convert's existing re-sign safety net never fired and a stale cached URL went straight to RVC (wrong/garbage voice; per-attempt variance came from flaky client-side F0 detection against the same dying link feeding different auto pitch-shifts into an otherwise deterministic RVC); (2) the music stems (bass/drums/other) had no durable paths at all and `mixStems`' `fetchDecode` swallowed failures, so expired music URLs produced a vocals-only mix that was marked "ready" and persisted as the full song. Fix, mirroring the existing `persistVocals` pattern: new shared `lib/stem-persist.ts` helper; stem-split now persists **all four** Demucs stems and returns `bassPath`/`drumsPath`/`otherPath`; karaoke-split persists lead/backing (`stems/<id>-lead.mp3`) and returns their paths (`maxDuration` 30→60); gender-split returns the `gender-split/<hash>-{male,female}.mp3` paths its `persistStems` already created; new auth-gated `/api/stems/refresh` batch re-signs stem paths (prefix-validated: `stems/`, `gender-split/`, `<uid>/`); `StemResult` carries all seven new path fields through the cache; the restore effect re-signs every stem URL via the new route **before** using the result and shows a visible "Restored …" toast; `handleProcess` sends the durable path matching whichever stem is actually converted (lead/male/female/full — single-job and dual-job branches) so the server-side re-sign now fires on every swap; and `mixStems` is strict — any failed stem fetch fails the whole mix into the visible 'error' state (auto-switching the player to Vocals-only) instead of silently shipping a partial mix. 5h cache TTL kept: re-signing on restore makes it safe. (commit: 277cae1)

## 2026-07-01

- Investigated two reported Warmth-control bugs ("slider does nothing" / "preview doesn't match saved file") — root-caused to the Polish panel being shown on both the Full-song and Vocals-only tabs while warmth was only ever wired into the Full-song `mixStems()` call. Vocals-only played `convertedVocalsUrl` directly for standard swaps, or a `mixStems()` blend built with no `{warmth}` opt for duet — so soloing the vocal (the natural way to judge an EQ tweak) produced zero change, and the saved file (always built from the Full-song render) differed from what had just been previewed. Fixed: the vocals-only blend effect now passes `debouncedWarmth` into `mixStems()` and depends on it (for both duet N>1 and single-vocal swaps), falling back to the instant raw `convertedVocalsUrl` only when there's nothing to blend or warm (single vocal, warmth=0). Also raised `WARMTH_MAX_DB` 6→10 dB so the low-shelf boost is clearly hearable instead of getting masked by the instrumental bed; warmth=0 still inserts no filter node (byte-identical). Found and deliberately left alone: an unrelated, currently-unreachable gap in the duet Mode 2/3 female-vocal RVC job (missing `protect`/`filterRadius`/`rmsMixRate` overrides) — logged in PROJECT_STATUS.md §6 for whenever duet fine-tuning ships. (commit: f0408c1)
- Honesty pass on the Configure/swap screen (`ConfigStep.tsx`) after auditing which of its six controls actually reach the backend — confirmed only Style Intensity (`index_rate`) and Pitch Shift are real; Age Range, Accent, and Output Language are never sent to any API. Rather than remove them (keeps the product vision visible), disabled all three (native `disabled` on the SegControl buttons / `<select>`s, greyed via CSS) and added a "Coming soon" badge next to each label. Separately, Gender Lock's status copy claimed a "hard guarantee" implying it constrains the *output voice's* gender — it doesn't; its only real effect is routing which duet stem gets converted, and only in single-singer duet mode. Reworded both status states (duet-split and no-split) to describe that actual behavior instead of an unmet guarantee; left the control itself enabled since the routing effect is real. Goal: nothing on Configure implies a capability the pipeline doesn't deliver. (commit: 5dbac6e)
- Add Reverb — the second effect in the vocal polish layer, chained after Warmth per the earlier investigation's design. `audioClip.ts` gained `createReverbImpulse()`, a synthetic exponentially-decaying noise buffer (~1.8s) generated in code — no bundled IR asset or network fetch. `mixStems()`'s vocal chain is now `gain → [warmth EQ if warm] → dry/wet split → destination`; the dry/wet split uses one shared `ConvolverNode` + dry/wet `GainNode` bus per call, fed by every vocal channel (convolution distributes over summed inputs, so sharing one convolver across duet channels is identical to giving each its own, just cheaper). reverb 0–100 maps to 0–35% wet — capped well under 100% since a fully-wet soloed vocal sounds washy/unnatural. At reverb=0 no convolver/split nodes are created at all, matching Warmth's byte-identical-at-zero guarantee. New debounced `reverb`/`debouncedReverb` state mirrors warmth's pattern exactly (280ms debounce, ref for the mount-time build, settle-gated persist); the warmth re-render effect, the vocals-only blend effect, and the persist-settle guard were all generalized to also key off reverb, so it gets the same both-tabs-plus-saved-file guarantee Warmth already has. New "Reverb" slider added next to Warmth in the existing Polish panel. No backend/API/dependency changes. (commit: 5c35648)

## 2026-06-30

- Recover clone-vocal quality — switch RVC `output_format` from `mp3` to `wav` in voice-convert (`route.ts:262`). The vocal was double-lossy-encoded (Demucs separation mp3 → RVC mp3) while the instrumental encodes only once, making the clone vocal sound duller than the music (~75% vs 95%). WAV removes that second compression; downstream (mixStems, Fine-tune preview) already decodes WAV. Single-variable change to measure impact alone — Demucs format untouched. Tradeoff: WAV is ~10× larger, so the vocal fetch in the browser mix/preview is slower. (commit: e5968d7)
- Add **Warmth** vocal-polish control to the Voice Swap result (first step of the polish/effects layer) — `ResultStep.tsx` only, no backend. `mixStems` gained an optional `{ warmth }`: the converted-vocal channel now routes through a Web Audio low-shelf BiquadFilter (200Hz, warmth 0–100 → 0–6 dB); the music bed is never filtered and warmth 0 inserts no node (mix byte-identical to before). New "Polish" UI block above the Fine-tune panel with one debounced (280ms) "Warmth" slider; a dedicated effect re-renders only the swapped full-song mix on change (no `fullMixState` flip, player stays usable). Because the parent persists exactly once (nulls its persist context after the first `onFullMixReady`), the single upload is now **deferred until warmth settles** (mix ready + not mid-drag + not re-rendering, +1s) so the saved Recent Swaps file reflects the chosen warmth. Known limit: re-adjusting warmth after that one settle updates the live preview but not the already-saved file (one-shot parent persist). (commit: 9b0100a)
- Fix Voice Lab Record-step layout regressions from the read-aloud script card: (1) record/stop button was clipped — `.vlrec-card` had no `flex-shrink`, so the flex-column workspace compressed it and its `overflow:hidden` clipped the bottom instead of scrolling; added `flex-shrink:0` (card keeps full height, workspace scrolls) and compacted the script card (smaller padding/font/line-height) so timer + button stay visible without scrolling. (2) Removed the now-duplicate instruction line in both panels (Quick "Speak naturally…" + Pro "Pick your audio interface…") since the script card replaces it; dropped their dead `.qr-hint`/`.pr-hint` styles. Timer, device label, and waveform/level meter untouched. (commit: b7887ed)
- Add a read-aloud script to the Voice Lab recording UI — users previously had no guidance on what to say while recording a clone. New `recordingScripts.ts` holds 3 phonetically-varied passages (~30–45s each, so one pass clears the 30s minimum) + a `nextScript()` rotor; RecordStep renders the active passage above the Quick/Pro capture panel (not Upload) under the label "Read this aloud (or sing across your range):" with a "New script" shuffle button. Includes explicit singing guidance (go low/high + soft/loud, don't read flatly) since clones here are often used for singing. Display-only — no backend/training changes. (commit: 61e7ad2)
